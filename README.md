# Прогнозирование стоимости домов в Калифорнии

Pet-проект по машинному обучению: линейная регрессия с дополнительной инженерией признаков для предсказания медианной стоимости дома на основе данных о жилье и демографии.

## Цель проекта

Построить модель, которая по таким параметрам, как доход населения, возраст дома, количество комнат, географические координаты и др., предсказывает медианную стоимость дома (`MedHouseVal`) в районах Калифорнии.

## Данные

Используется датасет **California Housing**, встроенный в библиотеку `sklearn` (загружается с помощью `pd.read_csv` по URL).

- **Обучающая выборка:** `train.csv` (2000 строк)
- **Тестовая выборка:** `test.csv` (без целевой переменной)

### Признаки

| Название     | Описание                                            |
|--------------|-----------------------------------------------------|
| `MedInc`     | Медианный доход в блоке                             |
| `HouseAge`   | Медианный возраст дома                              |
| `AveRooms`   | Среднее количество комнат в доме                    |
| `AveBedrms`  | Среднее количество спален в доме                    |
| `Population` | Население блока                                     |
| `AveOccup`   | Средняя заполняемость домохозяйства                 |
| `Latitude`   | Широта                                              |
| `Longitude`  | Долгота                                             |

Целевая переменная: `MedHouseVal` (медианная стоимость дома в сотнях тысяч долларов).

## Используемые библиотеки

- `numpy`, `pandas` – обработка данных
- `matplotlib`, `seaborn` – визуализация
- `scikit-learn` – предобработка, модели, пайплайны, метрики

## Этапы работы

### 1. Исследовательский анализ данных (EDA)
- Просмотр структуры данных (`head()`, `describe()`)
- Построение гистограмм распределений всех числовых признаков
- Визуализация зависимости цены от географического положения (scatter plot широта/долгота, цветом показана цена)

### 2. Предобработка
- Обрезка выбросов: для `AveRooms`, `AveBedrms`, `AveOccup`, `Population` установлены верхние границы (10, 3, 6, 5000 соответственно) – реализовано через собственный трансформер `Clamp`.
- Полиномиальные признаки второй степени (без включения смещения) – `PolynomialFeatures`.
- Стандартизация числовых признаков – `StandardScaler`.
- Кластеризация географических координат методом k‑means (100 кластеров) и one‑hot кодирование меток кластеров.

### 3. Моделирование
- Использован полный пайплайн `Pipeline`:
  1. `ColumnTransformer` – применяет разные преобразования к числовым и географическим признакам.
  2. `LinearRegression` – базовая линейная модель.
- Целевая переменная преобразована с помощью `log1p` (логарифмирование) для стабилизации дисперсии и улучшения нормальности остатков; прогноз возвращается обратно через `expm1`.

### 4. Оценка качества
- Разбиение обучающей выборки на train (90%) и validation (10%).
- Метрика: **RMSE** (Root Mean Squared Error).

Результаты:
- **Train RMSE:** 0.504
- **Test RMSE:**  0.480

### 5. Диагностика модели
- График остатков (`Residuals vs Fitted`) – визуальная проверка гомоскедастичности.
- График остатков против каждого признака.
- График `Actual vs Predicted` с линией идеального прогноза.

### 6. Прогноз на тестовых данных
- Предсказание для файла `test.csv` и сохранение результатов в `solution.csv`.

## Результаты и выводы

- Модель линейной регрессии с добавлением полиномиальных признаков и географических кластеров показала **RMSE ≈ 0.48** на валидационной выборке, что является хорошим результатом для данного датасета (базовый уровень иногда превышает 0.7).
- Графики остатков демонстрируют, что дисперсия остатков относительно постоянна, но присутствуют небольшие отклонения при низких и высоких прогнозах (что ожидаемо для линейной модели).
- Использование логарифмического преобразования целевой переменной улучшило нормальность распределения ошибок.
